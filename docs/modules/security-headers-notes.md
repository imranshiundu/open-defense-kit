# Security Headers Notes

## Purpose

This module is an educational checklist for defensive HTTP security headers.

It does not scan the internet or contact any website. It explains what to check on systems you own.

## Risk level

Low.

## Network activity

None.

## Common headers to understand

- `Content-Security-Policy`
- `Strict-Transport-Security`
- `X-Content-Type-Options`
- `X-Frame-Options`
- `Referrer-Policy`
- `Permissions-Policy`

## Safe use

Use this checklist when reviewing your own web apps, staging environments, or client systems where you have written permission.

## Future direction

A future version may include an authorized checker that requires explicit target ownership confirmation before making any request.
